第一部分：启动与基础检查

检查 1：服务能正常打开

操作：双击 start.bat，等待黑色命令行窗口出现文字，然后打开浏览器输入 localhost:8501。

通过标准：
[x] 浏览器显示三栏布局界面（左栏共识链、中栏候选区、右栏备弹区）
[x] 命令行窗口没有红色报错文字
[x] 整个过程 3 秒以内完成

失败标志：浏览器显示"无法访问此网站"、或命令行窗口出现红色报错并停止运行。


---

检查 2：飞书连接状态

操作：查看界面右上角或顶部的连接状态指示。

通过标准：
[] 显示"已连接"或绿色状态图标
[] 不显示"断开"或"轮询模式"


---

检查 3：备弹区 SKU 初始加载

操作：查看右栏备弹区，确认有内容显示。

通过标准：
[] 备弹区显示至少 3 条 SKU
[] 其中至少 1 条带有 🟢 或 🟡 标识
[] 每条 SKU 有标题和简短描述


---

第二部分：共识链基础操作

检查 4：快捷指令添加记录

操作：在输入框中依次输入以下三条指令，每条单独发送：

/记 毛利结构：储能系统集成毛利约18%，EPC工程毛利约12%
/记 交付情况：EPC项目平均周期6个月，无标准化交付流程
/记 资源分布：研发团队15人，销售团队8人，主要在华东区域

每条指令发送后，逐项确认：
[] 共识链区域出现绿色短暂闪动（约 0.3 秒）
[] 左栏完整度进度条数字上升
[] 记录出现在共识链列表，状态显示"待确认"


---

检查 5：关键词自动识别共识类型

操作：输入以下内容并发送：

/记 客户认可当前现金流压力较大

通过标准：
[] 这条记录与其他记录的样式明显不同（颜色、图标或标签有区别）
[] 说明系统识别出"客户认可"关键词，自动标记为共识类型


---

检查 6：确认一条记录

操作：找到第一条记录，输入以下指令（将 cc_001 替换为实际记录 ID）：

/确认 cc_001

通过标准：
[] 该记录状态变为"已确认"
[] 时间戳没有改变（确认操作不应更新时间）
[] 飞书多维表格"诊断共识"表中出现对应行（确认时间、发现内容字段有值）
[] 飞书"客户档案"表对应字段同步更新


---

检查 7：修正一条记录

操作：找到已确认的记录，使用修正指令：

/改 cc_001 毛利结构：储能系统集成毛利约20%，EPC工程毛利约12%（已核实）

通过标准：
[] 原来那条记录变成灰色删除线样式，状态变为"已作废"
[] 下方出现一条新记录，内容是修正后的内容
[] 原记录的内容没有被覆盖，仍然可以看到
[] 完整度进度条没有因为修正操作而下降


---

检查 8：阶段切换指令

操作：输入以下指令：

/切 商业模式

通过标准：
[] 界面阶段标签更新为"商业模式"
[] 之后新添加的记录，其阶段标注为"商业模式"（而非之前的阶段）


---

检查 9：完整度指示器

操作：查看界面上的完整度进度条或百分比数字。

通过标准：
[] 当前显示约 55%（5 个字段已填写）
[] 每新增一条与字段相关的已确认记录，数字会上升


---

第三部分：候选生成

检查 10：记录不足时的引导提示

操作：在共识链只有 1-2 条已确认事实的情况下，输入：

/候选

通过标准：
[] 系统不生成候选卡
[] 显示一段引导文字，提示需要更多已确认信息
[] 文案中包含"建议先追问"或类似表述，明确说明缺少什么


---

检查 11：正常生成 3 张候选卡

操作：确保共识链中有 3 条以上已确认事实，输入 /候选。

通过标准：
[] 1.5 秒以内弹出 3 张候选卡，横向并列显示，标注"候选A/B/C"
[] 3 张卡的风险标签各不相同（稳健、平衡、激进）
[] 3 张卡的文字长度大致相近，没有一张很长一张很短
[] 候选卡弹出后，共识链仍然可以看到，没有被完全遮挡


---

检查 12：选择候选方案后的状态

操作：按数字键 1 选择候选 A。

通过标准：
[] 候选 A 写入共识链，状态为"待客户确认"
[] 飞书"诊断共识"表不出现这条记录（待确认状态客户不可见）
[] 界面"待确认假设"区出现这条判断
[] 其他两张候选卡淡出但保留，没有消失


---

检查 13：确认候选方向

操作：输入 /确认。

通过标准：
[] 候选 A 状态变为"已确认"
[] 飞书"诊断共识"表出现这条记录，"建议方向"列有值
[] 界面"待确认假设"区该条目消失，移入"已确认"区


---

检查 14：缓存命中速度

操作：在没有修改任何记录的情况下，再次输入 /候选。

通过标准：
[] 这次几乎瞬间（< 0.2 秒）返回结果，明显比第一次快


---

检查 15：修正记录后缓存作废

操作：用 /改 修正任意一条共识记录，然后再次输入 /候选。

通过标准：
[] 这次返回时间回到正常速度（不再是瞬间），说明缓存被正确清除，系统重新计算

第四部分：演示模式（重点验收）

这是 Streamlit 版本的核心痛点，Node.js 版本必须完全解决，请逐项仔细验收。

检查 16：切换进入演示模式

操作：按键盘 Ctrl + Shift + D。

通过标准：
[] 0.1 秒以内切换完成，没有闪烁或白屏
[] 界面右上角状态徽标从 ⚫ 变为 🔵


---

检查 17：演示模式下内容正确隐藏（第一级）

通过标准（以下内容应该消失）：
[] Token 计数数字（如"已使用 1234 tokens"）
[] SKU 的 ID 编号（如"sku\_037"之类的技术标识）
[] 调试日志区域
[] 可信度的技术说明文字
[] 快捷指令提示栏（/记、/候选 等提示）


---

检查 18：演示模式下内容正确替换（第二级）

通过标准（以下内容应该被替换为客户友好表述）：
[] 阶段名（如"diagnosis"替换为"诊断阶段"等中文表述）
[] 完整度百分比替换为营销化表述
[] SKU 引用（如"\[新能源-sku\_037\]"替换为"标杆案例："）
[] "候选方案（预计算缓存命中）"替换为"初步建议方向"


---

检查 19：演示模式下视觉增强（第三级）

通过标准（以下内容应该被增强）：
[] 四栏状态板字体明显放大（约 120%）
[] 候选卡标题加粗加大
[] 知识库版本号移至显眼位置，格式变为"知识库v2.0 · XXX个新能源行业案例"
[] 右上角状态徽标位置正确


---

检查 20：演示模式下添加记录不退出（最关键）

操作：在演示模式下，继续输入一条新的记录：

/记 客户战略目标：3年内储能运营收入占比超过40%

通过标准：
[] 记录成功添加到共识链
[] 演示模式没有退出，界面仍然保持演示模式状态
[] 右上角 🔵 徽标仍然显示

这是 Streamlit 版本每次 rerun 都会重置演示模式的核心 Bug，Node.js 版本必须通过这一条。


---

检查 21：F11 快捷键也能切换

操作：先退出演示模式，然后按 F11 键。

通过标准：
[] F11 同样能触发演示模式切换
[] 效果与 Ctrl+Shift+D 完全一致

检查 22：退出演示模式

通过标准：
[] 所有隐藏和替换的内容全部恢复
[] 界面回到普通模式，状态徽标变回 ⚫

检查 44：服务重启后数据恢复

操作：

2. 关闭命令行窗口（强制停止服务）
3. 重新双击 start.bat 启动服务
4. 刷新浏览器
